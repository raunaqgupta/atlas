Faced with a backlog of captured [[Task]]s, the user needs to separate "today" from "not today" so they can start working with a finite, intentional list rather than thrashing across the whole pile. The job is recurring — usually daily — and is about decision-making, not task execution.

## Tasks

1. Open the unsorted/inbox view of [[Task]]s.
2. For each task, decide: today, this week, later, or cancel — updating the task's status or `due_at`.
3. Optionally tag with [[Tag]] for context (e.g. `#deep-work`, `#errand`) to enable later filtering.
4. Move resulting "today" set to a focused view and close the inbox.

## Success criteria

- The "today" set fits on one screen without scrolling.
- The user can name why each item is on the list.
